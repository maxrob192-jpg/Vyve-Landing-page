`landing` and `book` ‚Üí `maxrob192-jpg.github.io`. The main Wix website at
`www.vyvepsychotherapy.ca` is untouched.

## How booking works

Visitors browse these branded pages; every "Book" action deep-links into **Jane**
(new tab) with the right therapist and office pre-selected. Jane stays the single
source of truth and tracks all appointments.

```
https://clearviewpsychotherapy.janeapp.com/locations/<office-slug>/book#/staff_member/<id>
```

- Single-office therapists ‚Üí open that office's calendar directly.
- Dual-office therapists ‚Üí small "Choose an office" pop-up first.
- **Lisa Johnson Thebeau** is shown but marked *"Not accepting new clients"* (no booking).
- Jane **cannot be embedded in an iframe** (`X-Frame-Options: SAMEORIGIN`) ‚Äî the handoff
  to Jane's page is the supported pattern.

## Editing & publishing

Everything is generated by **`build_clean.py`** (in this folder):

- **Therapists / Jane IDs / offices** ‚Äî edit the `team` list at the top.
  Row: `(name, initials, role, modality, jane_id, [offices], accepting)`.
- **Copy / colors / spacing** ‚Äî copy lives in the page templates below the data;
  design tokens are in the `STYLE` block (`:root { ‚Ä¶ }`).
- Subdomain URLs are the `LANDING_URL` / `BOOK_URL` constants.

Publish flow:

```bash
python3 build_clean.py      # regenerates BOTH sites (this repo + ../vyve-book)
# then commit & push each repo (GitHub Desktop or git push)
```

GitHub Pages redeploys automatically ~1 minute after each push.
Hand-editing `index.html` directly also works ‚Äî just remember the two repos are
separate sites.
